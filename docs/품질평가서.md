# SI-Doc-Creator 품질 평가서

**평가 일시**: 2025-12-21
**평가자**: Antigravity (AI Agent)
**평가 대상**: SI-Doc-Creator (v1.0 Alpha)

---

## 1. 개요 (Executive Summary)

본 프로젝트는 SI 프로젝트 수행 시 반복되는 문서 작업을 자동화하기 위한 솔루션입니다. 현재 React 기반의 미려한 UI와 Node.js 기반의 문서 생성 엔진을 갖추고 있습니다. **UI/UX 측면에서는 상용 소프트웨어 수준의 높은 완성도**를 보여주나, **핵심 기능인 문서 생성 및 다국어 지원 부분에서 치명적인 결함과 미구현 사항**이 발견되었습니다.

**종합 등급**: **B-** (UI: A+, 기능: C, 안정성: B)

---

## 2. 프로세스 분석 (User Journey)

### 2.1 워크플로우 효율성
- **흐름**: [로그인] → [프로젝트 등록/분석] → [템플릿 관리] → [문서 생성]
- **평가**: 전체적인 흐름이 매우 직관적이며, 사용자가 무엇을 해야 할지 명확하게 유도합니다. 특히 `AnalyzePage`를 통한 기존 프로젝트 폴더 분석 기능은 사용자의 초기 데이터 입력 부담을 획기적으로 줄여주는 킬러 기능입니다.
- **아쉬운 점**: 문서 생성(`GeneratePage`) 후 생성된 파일을 다운로드하는 것 외에, 생성 이력을 관리하거나 재다운로드할 수 있는 기능이 부재합니다.

---

## 3. UI/UX 평가 (User Experience)

### 3.1 디자인 및 심미성 (**Excellent**)
- **비주얼**: 다크 모드 기반의 네온 그라데이션, 카드형 레이아웃, 부드러운 애니메이션 등 최신 트렌드를 매우 잘 반영했습니다. 
- **피드백**: 로딩 스피너, 토스트 메시지(추정), 툴팁 등 사용자 인터랙션에 대한 피드백이 적절합니다.

### 3.2 편의성 (**Good**)
- **AnalyzePage**: 로컬 폴더 브라우징 및 자동 분석 기능의 UI가 매우 훌륭합니다. 드라이브 목록 자동 로드, 폴더 탐색 등의 경험이 매끄럽습니다.
- **TemplatePage**: 파일 포맷(PPTX/DOCX)에 따른 색상 구분, 플레이스홀더 자동 감지 표시 등 세심한 배려가 돋보입니다.
- **GeneratePage**: 다수 템플릿 일괄 선택 및 생성 진행률 표시 기능이 우수합니다.

## 4. 기능 및 기술적 완성도 (Technical Quality)

이 부분에서 심각한 문제점들이 발견되었습니다.

### 4.1 다국어 지원 기능의 허구 (Critical Issue - "Fake Feature")
- **현상**: `HomePage`와 `GeneratePage` UI에는 한국어/영어/베트남어 선택 버튼이 존재하고, 마치 기능이 작동하는 것처럼 보입니다.
- **실체**: 클라이언트 API(`api.ts`) 및 서버 라우트(`documents.ts`)를 분석한 결과, **언어 설정(`language`) 파라미터가 서버로 전혀 전송되지 않고 있습니다.** 즉, UI만 존재하고 실제 기능은 구현되지 않았습니다.
- **영향**: 사용자는 다른 언어로 문서를 생성하려 해도 무조건 원본 템플릿 언어(주로 한국어)로만 생성됩니다. 이는 사용자 신뢰를 크게 떨어뜨릴 수 있습니다.

### 4.2 플레이스홀더 파싱 이슈 (Major Issue)
- **현상**: `templateParser.ts` (템플릿 업로드 시)와 `preprocessXml` (문서 생성 시) 간의 로직 불일치.
    1. **업로드 시**: `preprocessXml`을 거치지 않고 Raw XML에서 정규식 검색 → Word의 XML 파편화 문제로 인해 플레이스홀더를 감지하지 못함 (0개로 표시됨).
    2. **생성 시**: `preprocessXml`을 사용하지만 **단일 Run(`<w:r>`) 내에서만 병합**을 수행함. 플레이스홀더가 여러 Run에 걸쳐 쪼개진 경우(`{{` + `프로젝트명` + `}}`) 치환되지 않음.
- **해결 필요**: 파서 로직의 전면적인 개선이 필요합니다.

### 4.3 문서 생성 엔진
- DOCX/PPTX 생성은 `JSZip`과 정규식 치환 방식을 사용하여 가볍게 구현되었습니다. 
- PDF 생성은 `PDFKit`을 사용하는데, 한글 폰트(NotosansKR 등)가 서버에 내장되어 있지 않으면 한글이 깨지는 고질적인 문제가 PRD에 "알려진 이슈"로만 남아있고 해결되지 않았습니다.

## 5. 개선 권고 사항 (Recommendations)

### 우선순위 1: 기능 정상화 (Immediate Action)
1. **Fake Feature 제거 또는 구현**: 다국어 지원 기능을 구현할 계획이 없다면 UI에서 숨기거나, 구현한다면 `documentApi.generate`에 `language` 파라미터를 추가하고 서버에 번역 로직(또는 언어별 데이터 매핑)을 추가해야 합니다.
2. **플레이스홀더 파서 개선**:
    - `templateParser.ts`에 `preprocessXml` 로직 적용.
    - `preprocessXml` 함수를 개선하여 Run 간 경계를 넘어서는 텍스트 병합 지원.
3. **PDF 한글 폰트 적용**: 서버 assets에 오픈 소스 한글 폰트를 포함시키고 PDF 생성 시 로드하도록 수정.

### 우선순위 2: 사용자 경험 강화
1. **다운로드 이력 관리**: 생성된 문서의 이력을 저장하고 다시 다운로드할 수 있는 기능 추가.
2. **템플릿 미리보기**: 업로드한 템플릿의 플레이스홀더 위치를 미리 볼 수 있는 기능.

### 우선순위 3: 아키텍처 개선
1. **문서 생성 엔진 고도화**: 단순 문자열 치환 방식은 복잡한 서식이나 반복문(리스트, 표) 처리에 한계가 있으므로, `docxtemplater` 같은 전문 라이브러리 도입 고려.

---

## 6. 결론

SI-Doc-Creator는 **"보기 좋은 떡"**이지만, **"속이 덜 익은"** 상태입니다. UI의 완성도가 높아 사용자의 기대치를 높이지만, 실제 문서를 생성해보면 다국어 미지원이나 치환 실패 등으로 실망감을 줄 수 있습니다. 

현재의 UI 품질을 유지하면서 **Back-end의 문서 생성 로직을 단단하게 보강**한다면, 실제 현업에서 매우 유용하게 쓰일 수 있는 훌륭한 도구가 될 것입니다.
