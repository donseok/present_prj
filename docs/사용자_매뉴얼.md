## 1. 소개

### 1.1 DocuGen이란?
DocuGen(다큐젠)은 SI(System Integration) 프로젝트를 위한 문서 자동 생성 시스템입니다. 프로젝트 정보를 입력하고 템플릿을 선택하면, 전문적인 문서를 자동으로 생성해줍니다.

### 1.2 주요 기능
- **문서 자동 생성**: 프로젝트 정보 기반 문서 자동 생성
- **다양한 템플릿**: 요구사항정의서, 수행계획서, WBS 등 10종의 문서 템플릿
- **다중 출력 형식**: PPTX, DOCX, PDF 형식 지원
- **다국어 지원**: 한국어, 영어, 베트남어 지원
- **프로젝트 관리**: 프로젝트 정보 등록 및 관리
- **템플릿 관리**: 사용자 정의 템플릿 업로드 및 관리

## 3. 로그인

### 3.1 로그인 화면 접속
웹 브라우저에서 DocuGen URL에 접속하면 로그인 화면이 표시됩니다.

### 3.2 로그인 방법
1. **아이디** 입력란에 사용자 아이디를 입력합니다
2. **패스워드** 입력란에 비밀번호를 입력합니다
3. **로그인** 버튼을 클릭합니다

### 3.3 초기 계정 정보
- 아이디: `admin`
- 패스워드: `admin123`

### 3.4 로그아웃
화면 우측 상단의 사용자 아이콘 옆 **로그아웃** 버튼을 클릭하면 로그아웃됩니다.

| 출력 형식 | PPT/DOC/PDF 형식 선택 |
| 생성 버튼 | 문서 생성 실행 |

## 5. 문서 생성하기

### 5.1 문서 생성 단계

#### Step 1: 문서 제목 입력
- 생성할 문서의 제목을 입력합니다
- 예: "동국제강 공조설비 정비관리 시스템"

#### Step 2: 프로젝트 선택 (선택사항)
- 등록된 프로젝트가 있는 경우 선택합니다
- 프로젝트를 선택하면 해당 프로젝트 정보가 문서에 자동 반영됩니다
- 프로젝트를 선택하지 않으면 데모 모드로 동작합니다

#### Step 3: 문서 언어 선택
마우스를 올리면 툴팁이 표시됩니다:
- 🇰🇷 **한국어**: 한국어로 문서 생성
- 🇺🇸 **English**: 영어로 문서 생성
- 🇻🇳 **Tiếng Việt**: 베트남어로 문서 생성

#### Step 4: 템플릿 선택
생성할 문서 종류를 선택합니다 (단일 선택):

| 템플릿 | 설명 |
|--------|------|
| 요구사항정의서(PRD) | 기능/비기능 요구사항 정의 |
| 프로젝트 수행 계획서 | 프로젝트 개요, 일정, 팀 구성 |
| WBS | 작업분해구조, 마일스톤 |
| 기본설계서 | 시스템 설계, 아키텍처, ERD |
| 상세설계서 | 화면설계, 인터페이스 정의 |
| 중간보고서 | 진행현황, 이슈, 계획 |
| 테스트결과서 | 테스트 수행 결과 |
| 종료보고서 | 프로젝트 수행 결과 |
| 사용자 매뉴얼 | 시스템 사용자 가이드 |
| 운영자 매뉴얼 | 시스템 운영자 가이드 |

#### Step 5: 출력 형식 선택
마우스를 올리면 툴팁이 표시됩니다:
- **PPT (.pptx)**: PowerPoint 프레젠테이션
- **DOC (.docx)**: Word 문서
- **PDF (.pdf)**: PDF 문서

#### Step 6: 문서 생성
- **문서 생성하기** 버튼을 클릭합니다
- 생성 중에는 로딩 애니메이션이 표시됩니다

### 5.2 생성된 파일 다운로드
- 문서 생성이 완료되면 **생성된 파일** 섹션에 파일이 표시됩니다
- 파일 우측의 **다운로드** 버튼(⬇)을 클릭하여 다운로드합니다

## 6. 프로젝트 관리

### 6.1 프로젝트 목록 보기
상단 메뉴에서 **프로젝트**를 클릭하면 등록된 프로젝트 목록이 표시됩니다.

### 6.2 프로젝트 등록
1. **새 프로젝트** 버튼을 클릭합니다
2. 프로젝트 정보를 입력합니다:
   - 프로젝트명
   - 고객사
   - 설명
   - 범위
   - 시작일 / 종료일
   - 상태
3. 팀원 정보를 추가합니다
4. 마일스톤을 설정합니다
5. 요구사항(기능/비기능)을 입력합니다
6. **저장** 버튼을 클릭합니다

### 6.3 프로젝트 수정
1. 프로젝트 목록에서 수정할 프로젝트를 선택합니다
2. **수정** 버튼을 클릭합니다
3. 정보를 수정한 후 **저장** 버튼을 클릭합니다

### 6.4 프로젝트 삭제
1. 프로젝트 목록에서 삭제할 프로젝트를 선택합니다
2. **삭제** 버튼을 클릭합니다
3. 확인 메시지에서 **확인**을 클릭합니다

---

## 7. 템플릿 관리

### 7.1 템플릿 목록 보기
상단 메뉴에서 **템플릿**을 클릭하면 등록된 템플릿 목록이 표시됩니다.

### 7.2 템플릿 업로드
1. **템플릿 업로드** 버튼을 클릭합니다
2. 업로드할 파일을 선택합니다 (DOCX 또는 PPTX 형식)
3. 템플릿 정보를 입력합니다:
   - 템플릿명
   - 문서 유형
4. **업로드** 버튼을 클릭합니다

### 7.3 플레이스홀더 사용법
템플릿 내에서 `{{플레이스홀더}}` 형식을 사용하면 문서 생성 시 자동으로 값이 대체됩니다.

| 플레이스홀더 | 설명 |
|--------------|------|
| `{{프로젝트명}}` | 프로젝트 이름 |
| `{{고객사}}` | 고객사 이름 |
| `{{설명}}` | 프로젝트 설명 |
| `{{범위}}` | 프로젝트 범위 |
| `{{시작일}}` | 프로젝트 시작일 |
| `{{종료일}}` | 프로젝트 종료일 |
| `{{상태}}` | 프로젝트 상태 |
| `{{팀원수}}` | 팀원 수 |
| `{{마일스톤수}}` | 마일스톤 수 |
| `{{팀원목록}}` | 팀원 목록 |
| `{{마일스톤목록}}` | 마일스톤 목록 |

### 7.4 템플릿 삭제
1. 템플릿 목록에서 삭제할 템플릿을 선택합니다
2. **삭제** 버튼을 클릭합니다
3. 확인 메시지에서 **확인**을 클릭합니다

## 8. FAQ

### Q1: 문서 생성 버튼이 비활성화되어 있습니다.
**A**: 문서 제목과 템플릿을 모두 선택해야 생성 버튼이 활성화됩니다.

### Q2: 프로젝트를 선택하지 않고 문서를 생성하면 어떻게 되나요?
**A**: 데모 모드로 동작하며, 플레이스홀더가 샘플 데이터로 대체됩니다. 실제 프로젝트 정보가 반영된 문서를 생성하려면 프로젝트를 먼저 등록하고 선택해야 합니다.

### Q3: 한 번에 여러 템플릿을 선택할 수 있나요?
**A**: 아니요, 현재 버전에서는 한 번에 하나의 템플릿만 선택할 수 있습니다.

### Q4: PDF 문서에서 한글이 깨져 보입니다.
**A**: 현재 PDF 생성 시 기본 폰트를 사용하고 있어 한글이 제대로 표시되지 않을 수 있습니다. 이 경우 DOCX 형식으로 생성 후 PDF로 변환하는 것을 권장합니다.

### Q5: 사용자 정의 템플릿은 어떤 형식을 지원하나요?
**A**: DOCX(Word) 및 PPTX(PowerPoint) 형식을 지원합니다.
